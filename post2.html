Networking Lab Walkthrough: Understanding Subnetting with CIDR Notation

Objective

CIDR (Classless Inter-Domain Routing) notation is a method used to specify IP addresses and their associated network prefixes. It allows for flexible allocation of IP addresses without being bound to traditional classful boundaries.

In CIDR notation, an IP address is followed by a forward slash and a number representing the number of significant bits in the network prefix. For example, 192.168.2.0/26 indicates that the first 26 bits are the network portion.

Task 1: Determining Subnet Mask and Subnet Bits

We start with an IP address (192.168.2.0) and a given subnet prefix (/26). By adding 2 bits to the standard 24-bit subnet mask (255.255.255.0), we calculate a new subnet mask of 255.255.255.192.

This new subnet mask corresponds to borrowing 2 bits for subnetting, leaving 6 bits for host addresses.

Task 2: Calculating Subnets and Hosts

With 2 bits borrowed for subnetting (/26), we calculate the number of subnets and hosts. Borrowing 2 bits for subnetting gives us 2^2 = 4 subnets.

The remaining 6 bits are available for host addresses, giving us 2^6 = 64 hosts per subnet.

Task 3: Identifying the Subnet for a Given Host

We encounter a host IP address (172.16.100.118) with a subnet prefix (/29). Examining the subnet mask, we see that /29 corresponds to 255.255.255.248. By counting 5 bits into the last octet, we note that subnets increase in increments of 8.

We quickly identify the subnet containing the host by jumping in increments of 8 and find that 172.16.100.112 is the subnet containing the host 172.16.100.118.

Conclusion

Through this lab, we've gained practical experience in subnetting IPv4 addresses using CIDR notation. We've learned how to calculate subnets, determine the number of hosts per subnet, and identify the subnet for a given host address. This skill is essential for network administrators and engineers to efficiently manage IP address allocation and routing.
