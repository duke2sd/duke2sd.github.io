Networking Lab Walkthrough: Configuring MAC Address Filtering on a Switch

Lab Walkthrough: Configuring MAC Address Filtering on a Switch

Lab Walkthrough: Configuring MAC Address Filtering on a Switch Using Packet Tracer

Lab Objective:

Learn how to configure MAC address filtering on a switch to control which devices can connect through specific ports.

Lab Purpose:

MAC address filtering is a security measure to ensure only authorized devices can access the network through a switch port. This is typically implemented via switch port security settings.

Lab Tool:

  • Cisco Packet Tracer

Lab Topology:

  • Two PCs connected to a hub
  • Hub connected to a switch

Steps:

Task 1: Connect Devices

  1. Open Packet Tracer and set up the topology as described.
  2. Connect a hub to a switch using a crossover cable from the hub to F0/1 on the switch.
  3. Connect two PCs to the hub using straight-through cables.

Task 2: Configure IP Addresses

  1. Click on PC0.
  2. Go to the Config tab.
  3. Select Fast Ethernet0.
  4. Set the IP Configuration to Static.
  5. Configure the following:
    • IPv4 Address: 192.168.1.1
    • Subnet Mask: 255.255.255.0
  6. Repeat the above steps for PC1 with the following settings:
    • IPv4 Address: 192.168.1.2
    • Subnet Mask: 255.255.255.0

Task 3: Note the MAC Address of PC1

  1. Click on PC1.
  2. Go to the Config tab and select Fast Ethernet0.
  3. Copy the MAC Address (e.g., 0060.47BB.C0E3).

Task 4: Configure MAC Address Filtering on the Switch

  1. Click on the switch and go to the CLI tab.
  2. Enter the following commands to configure port security: 
    Switch>enable
Switch#configure terminal
Switch(config)#interface fastEthernet 0/1
Switch(config-if)#switchport mode access
Switch(config-if)#switchport port-security
Switch(config-if)#switchport port-security mac-address 0060.47BB.C0E3
Switch(config-if)#end

    Ensure to replace the MAC address with the one you copied from PC1.

Task 5: Verify Port Security Settings

  1. On the switch CLI, enter the command: 
    Switch#show port-security interface fastEthernet 0/1

    The output should show the port security settings.

Task 6: Test Connectivity with Ping

  1. From PC0, open the Command Prompt.
  2. Ping PC1 using the command: 
    ping 192.168.1.2

    This action will test the MAC address filtering by sending frames through the switchport via the hub.

Task 7: Check Port Security Status

  1. On the switch CLI, enter the command: 
    Switch#show port-security

    The interface should now show as down due to a security violation.

Task 8: Verify MAC Address Triggering the Shutdown

  1. Check the MAC address of PC0 to confirm it was the device that caused the port shutdown.

Conclusion:

This lab demonstrates how to configure MAC address filtering on a switch to control network access. By setting up port security, only authorized devices (identified by their MAC addresses) are allowed to communicate through specific switch ports. This enhances network security by preventing unauthorized access.

Key Takeaways:

  • Understanding how to configure and verify port security on a switch.
  • Learning the importance of MAC address filtering in securing network ports.
  • Gaining practical experience with Packet Tracer for network configuration and troubleshooting.
