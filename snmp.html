Networking Lab Walkthrough: Configuring SNMP on a Cisco Router

Task 1: Configure the Router's IP Address

1. Enter configuration mode on the router:

Router(config)#int f0/0

2. Assign the IP address:

Router(config-if)#ip add 192.168.1.1 255.255.255.0

3. Bring the interface up: Router(config-if)#no shut Task 2: Configure the PC's IP Address 1. On the PC, navigate to the `IP Configuration` window. 2. Set the following: - IP Address: 192.168.1.2 - Subnet Mask: 255.255.255.0 - Default Gateway: 192.168.1.1 Task 3: Configure SNMP on the Router 1. Set up SNMP read/write access with the community string `101labs`: Router(config)#snmp-server community 101labs rw You should see a message indicating a warm start. Task 4: Open the MIB Browser on the PC 1. Open the MIB browser on the PC. 2. Enter the router's IP address (192.168.1.1). 3. In the ‘Advanced’ settings, input `101labs` for both read and write community strings. Task 5: Access the sysName MIB 1. In the MIB browser, drill down to `.iso.org.dod.internet.mgmt.mib-2.system.sysName`. 2. Press `GO` to populate the router's current name, which should be `Router`. Task 6: Change the Router Name Using SNMP 1. In the MIB browser, change the operation to `Set`. 2. Set the data type to `OctetString`. 3. Enter `101Labs` as the value. 4. Press `OK`, then press `GO`. Task 7: Verify the Router Name Change 1. You should see the router's name change to `101Labs`. 2. On the router, press `Enter` to confirm the change. Notes: SNMP allows not only network monitoring but also configuration changes. Most SNMP software includes a GUI for easier device management.
